# Инструкции для ИИ-ассистента — проект «Конструктор»

Читай этот файл **перед любой правкой** `index.html`.

---

## Обязательно после каждого изменения сайта

После внесения любого значимого изменения ИИ **сам** обязан:

1. **Увеличить версию** в двух местах `index.html`:
   - Константа в JS: `const CURRENT_VERSION = 'v2.XX';`
   - Отображение в шапке HTML: `>v2.XX</div>` (ищи рядом с «Разработал Лингидан»)

2. **Добавить запись в начало массива `CHANGELOG`** в JS:
   ```js
   {
     version: 'v2.XX', date: 'ДД.ММ.ГГГГ',
     changes: [
       'Краткое описание изменения',
       'Ещё одно изменение если есть',
     ]
   },
   ```

3. **Правила описания изменений:**
   - 1–3 пункта на версию
   - Без технических деталей: «Добавлен пикер исполнителей» — да, «изменён CSS-класс .cpm-item» — нет
   - Формат: что добавлено / что исправлено / что улучшено

**Почему:** пользователь не будет вручную вести историю обновлений — это задача ИИ.

## Архитектура проекта

- **Один файл** `index.html` (~4300 строк) — весь HTML + CSS + JS
- **Без сборщика, без backend** — чистая ваниль
- **Хостинг:** GitHub Pages → `https://lingidan.github.io/konstruktor/`
- **Хранение данных:** `localStorage` браузера
- **Генерация документов:** библиотека `docx` (подключена через CDN)
- **База сотрудников:** Google Sheets CSV

## Ключевые переменные

| Имя | Где | Описание |
|-----|-----|----------|
| `CURRENT_VERSION` | JS ~строка 3060 | Текущая версия — менять при каждом обновлении |
| `CHANGELOG` | JS перед `// Init` | Массив истории обновлений — макс. 10 записей |
| `currentUser` | JS | Авторизованный пользователь (null если не вошёл) |
| `tabsMeta` | JS | Метаданные вкладок `{tabs, activeId}` |

## Ключевые localStorage-ключи

- `narad_tabs_meta` — метаданные вкладок
- `narad_form_tab_{N}` — данные формы вкладки N
- `narad_theme` — выбранная тема
- `auth_user` — сохранённая сессия пользователя

## Стиль работы с Максимом

- Он пишет **голосом** — читай смысл, не застревай на опечатках
- Не объясняй каждую строку кода
- Не спрашивай подтверждения на мелкие правки — просто делай
- Для крупных фич: уточни детали → получи «ок» → реализуй
- **Нет git в папке** → нельзя использовать параллельных агентов с воркдеревьями

## Проверка версии при скачивании

Функция `ensureActualVersion()` сравнивает версию локального файла с prod.
Сравнивает **числами** (не строками) через `_parseVer()` — перезагружает страницу
только если удалённая версия **новее** текущей. Не ломай эту логику.
